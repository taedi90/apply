고려해볼 사항
- 환율 정보를 담을 객체 필요여부
- 환율 정보를 static으로 보관할 경우의 문제점? 변동주기?
- 환율 정보를 API에서 받지 못했을 경우의 예외처리
- API 키 암호화 처리
- submit -> ajax 처리 고려해보기(화면 전환 방지)
- 콤보박스를 변경했을 때 결과 지우기
- 환율 * 송금액 -> 수취금액

- 환율 소수점 2째자리까지, 정수부분 콤마처리
- 수취금액 미입력, 0미만 또는 10,000USD 초과, 숫자 외의 값 -> "송금액이 바르지 않습니다." - red color
